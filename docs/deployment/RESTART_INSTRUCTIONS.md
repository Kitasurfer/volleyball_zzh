# Инструкции по перезапуску для работы с локальным чатботом

## Что было сделано:

1. ✅ Установлен Supabase CLI
2. ✅ Создан локальный сервер для функции chatbot на порту 54321
3. ✅ Сервер запущен: `http://localhost:54321/functions/v1/chatbot`
4. ✅ Обновлён `.env.local` для использования локального URL
5. ✅ Создана резервная копия: `.env.local.backup`

## Что нужно сделать:

### Перезапустить Vite сервер:

1. Перейдите в терминал где запущен Vite (процесс 39991)
2. Нажмите `Ctrl+C` чтобы остановить сервер
3. Запустите снова: `pnpm dev` или `npm run dev`

### Проверить работу:

1. Откройте приложение в браузере
2. Откройте чатбот (кнопка в правом нижнем углу)
3. Задайте вопрос
4. Чатбот теперь должен работать через локальный сервер

## Логи локального сервера:

Локальный сервер chatbot запущен в фоновом режиме (процесс 70494).
Чтобы увидеть логи, используйте команду:

```bash
lsof -i :54321
```

## Вернуться к продакшн серверу:

Если нужно вернуться к использованию продакшн Supabase:

```bash
cd /Users/bogdanfesenko/StudioProjects/Volleyball/volleyball_zzh
cp .env.local.backup .env.local
```

Затем перезапустите Vite сервер.

## Остановить локальный сервер chatbot:

```bash
kill 70494
```

Или найдите процесс:
```bash
lsof -i :54321
kill <PID>
```
