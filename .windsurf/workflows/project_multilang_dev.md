## project_multilang_dev Workflow

1. **Разведка задачи**
Отвечай на русском языке.
   - Прочитай актуальные правила: `.windsurf/rules.md` и профильные `AGENTS.md`.
   - Определи затронутые модули (страницы, голосовой ассистент, admin, Supabase и т.д.).
   - Выпиши, какие языки/переводы и ассеты придется обновлять.

2. **Подготовка окружения**
   - Убедись, что стоят зависимости: `pnpm install` (при необходимости).
   - Запусти локально `pnpm dev` и, если нужна прокся/бэкенд, подними сопутствующие сервисы (например, Chatterbox) согласно документации.
   - При фронтовых правках открой страницу и отследи текущее состояние.

3. **Работа с текстами и данными**
   - Все новые строки добавляй/обновляй сразу в четырёх языках (`de`, `en`, `ru`, `it`) в соответствующих `content`-объектах или `src/data/translations.ts`.
   - При изменении структур перевода обновляй типы (`Translation`, интерфейсы компонента).
   - Для ассетов обновляй `public/images/image_meta.json` (url, caption, размеры).

4. **Реализация**
   - Соблюдай модульность: UI в `components/ui`, логика в hooks/services.
   - Для голосового ассистента следуй `src/components/chatbot/voice/AGENTS.md` (Web Speech API + Chatterbox, обязательные cleanup).
   - Стили пиши через Tailwind, уважай текущую дизайн-систему.
   - Не оставляй `console.log`/неиспользуемые импорты.

5. **Тесты и проверки**
   - Локально прогоняй минимум `pnpm lint`, `pnpm test`, `pnpm build` (для объёмных задач — профильные workflows `/voice_*`).
   - Проверяй UI на desktop и mobile (≈375px) и на всех языках.
   - Убедись, что Seo-метаданные и JSON-LD актуальны после замены изображений.

6. **Документация и фиксация**
   - При необходимости обнови `docs/` (архитектура, API, voice-инструкции).
   - Опиши изменения (что обновлено, какие языки и тесты) в итоговом отчёте/PR.

7. **Финальная валидация**
   - Просмотри diff на предмет побочных изменений.
   - Проверь, что workflows/правила соблюдены и нет временных файлов.
   - Останови локальные сервисы и подготовь отчёт пользователю.
